# Rewrite mk_entry.cpp from stubs to real module orchestration

The entry point has been gutted and rebuilt. Where 530 lines of stub classes lived—each reimplementing module logic inline—there's now an 11-module `#include` tree and a single `MKSystem` struct that holds all real instances. The REPL is fully wired: `/ask`, `/search`, `/think`, `/weather`, `/time`, `/news`, `/learn`, plus natural language routing through `MKSmartRouter` with fallback chains. Shutdown persists memory and improvement logs.

**Watch for:** Command parsing silently drops bare commands like `/ask` into "Unknown command" instead of showing usage (confirmed). The INSTANT route in `handle_natural_query` relies on naive `std::string::find` for city extraction from natural language, which will misfire on anything beyond "weather in X" (confirmed). The `MKReasoningChains` module is instantiated but never called—dead weight with startup cost (confirmed).

## High-level view

The `#ifndef` guards across all `.cpp` files make the transitive include tree safe despite `deep_reasoner.cpp` internally including `pattern_graph.cpp`, and `knowledge_integrator.cpp` pulling in `search_engine.cpp` which pulls in `http.cpp`.

The REPL dispatch uses `input.size() > N && input.substr(0, N)` for command parsing, which creates a gap: typing `/ask` without a trailing space and argument falls through to "Unknown command" rather than the usage message. This applies to all argument-bearing commands.

Natural language input goes through `MKSmartRouter`'s keyword scoring, which picks a route (INSTANT, GRAPH, SEARCH, REASON, GENERATE) and an ordered fallback chain when confidence is low. The INSTANT route attempts to extract structured parameters (city, timezone) from free text but does so naively—the time sub-path ignores what the user actually said and always fetches "America/New_York".

Shutdown is two calls: `memory.saveToDisk()` and `improver.saveLog()`. Newly learned facts are flushed on each `/learn` via `persistNewFact`, but session interaction data has no mid-session persistence, making it vulnerable to unclean kills.

<summary>Issues (5)</summary>

1. **Bare command falls to "Unknown command"** — `/ask`, `/search`, `/learn`, `/weather`, `/think` without arguments hit the else branch instead of displaying usage. Add equality checks (e.g., `input == "/ask"`) that call the handler with an empty string.
2. **INSTANT route city extraction is fragile** — `handle_natural_query` searches for " in " in the query and takes everything after it. "What's the weather in New York City today" yields "New York City today". Consider stripping trailing noise or accepting that the API will reject bad city names gracefully.
3. **`chains` member is unused** — `MKReasoningChains chains` is constructed (loading rules from disk) but never called anywhere in the REPL or natural language handler. Either wire it into the REASON route or remove it to avoid startup cost.
4. **No periodic auto-save** — If the process is killed (SIGKILL, OOM, crash), all session memory and improvement data since boot is lost. A periodic flush (even every 50 interactions) would limit the blast radius.
5. **`/time` default timezone is hardcoded** — `cmd_time` defaults to "America/New_York" with no way for the user to configure it. The INSTANT route's time sub-path has the same problem: it ignores timezone mentioned in natural language.

## Command dispatch gap on bare commands

The REPL checks commands with patterns like `input.size() > 5 && input.substr(0, 5) == "/ask "` (note the trailing space). If a user types just `/ask` or `/think`, the size check fails (4 and 6 characters respectively; `/think` is exactly 6 which fails `> 7`). The input falls through every condition to the final else:

```cpp
} else {
    std::cout << "\n  Unknown command. Type /help for options.\n";
}
```

Each `cmd_*` function already handles the empty-argument case with a usage message. The fix is adding equality checks before or after the sized checks.

## INSTANT route natural language parsing

The weather path searches for `" in "` and takes the remainder as the city:

```cpp
size_t inPos = lower.find(" in ");
if (inPos != std::string::npos) {
    city = input.substr(inPos + 4);
}
```

For "what's the weather in New York this week" it sends "New York this week" to the geocoding API. Worse, the time path doesn't attempt timezone extraction at all—if someone asks "what time is it in Tokyo", it fetches New York time and reports that at 0.95 confidence. The asymmetry between weather (attempts extraction, gets trailing noise) and time (no extraction, wrong answer) means the INSTANT route will produce confidently wrong answers for time-related queries that mention a location.

## Dead `MKReasoningChains` member

`MKSystem` constructs `MKReasoningChains chains("ai_core/hre/knowledge_files")`, which on construction calls `loadBuiltinRules()` and prints to stdout. But the REASON route goes directly to `sys.reasoner.think(input, sys.graph)` (the `MKDeepReasoner`). The `MKReasoningChains` module provides rule-based inference (inheritance, transitivity) which is complementary to `MKDeepReasoner`'s multi-hop traversal—it was likely intended for integration but not wired up.

## Shutdown durability

If the process dies without reaching shutdown (SIGKILL, segfault, OOM), all accumulated interactions and improvement data for the session is lost. The `dirty` flag in `MKPersistentMemory` exists but is never checked mid-session. A periodic save after N interactions would provide crash resilience without significant I/O cost given the flat-file format.
