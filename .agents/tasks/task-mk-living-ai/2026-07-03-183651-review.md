## High-level view

The SSH controller's `shellEscapeSingleQuotes` handles the standard `'\''` idiom and is applied where remote commands enter the shell string. The SCP builder (`buildSCPCommand`) interpolates file paths without any escaping—these paths come from the file-transfer API rather than user-typed REPL input, reducing but not eliminating the attack surface.

Device-comm's HMAC validation is structurally sound: `generateAuthToken` produces `HMAC-SHA256(secret, body)`, `verifyAuthToken` uses constant-time byte comparison, and `validateRequest` is exposed for inbound request handling. A legacy timestamp-window overload uses standard `==` comparison instead of constant-time, but that code path only triggers for GET requests with empty bodies.

The autonomous learner's `learnFromWeb` pipeline is wired into the daemon job with fact validation through `MKKnowledgeValidator` before graph integration. The `knowledge_sync` job, by contrast, remains a stub—it checks queue size without ever calling `getFactsForPeer`, `serializeQueue`, or pushing data to registered peers. Multi-device sync is architecturally present but the outbound push never fires.

## SCP path injection in `ssh_controller.cpp`

The fix at line 77 correctly applies `shellEscapeSingleQuotes(remote_cmd)` inside the SSH command builder. However, `buildSCPCommand` at line 83 constructs:

```cpp
cmd << " " << src_config.user << "@" << src_config.ip << ":" << src_path;
cmd << " " << dst_config.user << "@" << dst_config.ip << ":" << dst_path;
```

Neither `src_path` nor `dst_path` is quoted or escaped. A path containing spaces, semicolons, or backticks would break the command or inject code. The file-transfer method `transferFile` is the only caller, so the blast radius depends on where path values originate—if they ever come from user input, device names, or synced knowledge facts, injection is possible.

## Knowledge sync job wired to a read-only call

The daemon registers:

```cpp
sys.daemon.addJob("knowledge_sync", 300, [&sys]() {
    (void)sys.knowledgeSync.queueSize();
});
```

`MKKnowledgeSync` has methods for the full pipeline: `getFactsForPeer`, `serializeQueue`, `markSynced`, and `setPeerConnected`. None are called from the daemon. Outbound push to peers never occurs, meaning knowledge accumulated on one device will not propagate to others without manual intervention.

## Autonomous learner: dead method with wrong path

`learnFromWeb` invokes `python3 python/web_scraper.py` (correct path). The legacy `generateScraperCommand` references `python3 tools/web_scraper.py` (non-existent directory). If any future code calls the legacy method, it will silently fail. The dead method should be removed.
